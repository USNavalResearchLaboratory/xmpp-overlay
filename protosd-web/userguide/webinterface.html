ProtoSD Web Interface

ProtoSD has a dashboard for easy navigation of all of the functionality exposed through the Web interface, as follows:

As the screenshot shows, using the dashboard you can create a new mDNS service advertisement, create a new service discovery instance (for JmDNS or INDI), view or update an existing service discovery instance (to chnage the discovery parameters for example), view the discovered services in a tree, view analysis results from this discovery instance or view the documentation.

To create a new service advertisement, the following information has to be entered:

  • Service Name: a human-readable name that uniquely describes the service.
  • Type: a string of the form "_applicationprotocol._transportprotocol".
  • Domain: You either pass "local" here or a specific string if you have some particular reason to want to register in some specific remote domain.
  • Port:the port number of the service
  • TXT: The specific nature of the TXT record and how it is to be used is service type dependent.
